Security Policy

Protecting your information, accounts, and application data.

Last Updated: October 7, 2025
🛡️ High Protection

Security is at the core of how we design, build, and run the Pravin Tamilan service. We implement industry-standard safeguards to keep your personal data and account identifiers secure.

🔑

1. Authentication & Identity Protection

We use Google Sign-In protocol for authenticated interactions. Your Google authentication details are routed directly through Google's secure OAuth 2.0 servers. Pravin Tamilan never reads, logs, or stores your passwords, ensuring credentials can never be leaked or intercepted.

🔒

2. Data Transmission Encryption

All network communication between your device and our server/Firebase instances is secured using Transport Layer Security (TLS/HTTPS). This prevents eavesdropping and tampering with data while in transit.

💾

3. Local Device & Cache Security

Preferences, wallpapers, bookmarks, and user settings are stored locally on your device within sandbox containers provided by the Operating System (Android/iOS sandbox or browser IndexedDB). These local files are isolated and cannot be read by other applications on your device.

☁️

4. Cloud Database Protection

Cloud components utilize secure Firebase security rules configured to allow verified write-access only to authorized accounts. Database tables are monitored and regularly audited for security compliance and credential safety.

🔍

5. Vulnerability Disclosures

We welcome vulnerability reports from users and independent security researchers. If you identify a potential security issue in the Pravin Tamilan application, web code, or APIs, please report it immediately to us at apravint@gmail.com. We will investigate the issue and address it in accordance with standard patch releases.
